# 出力形式（mdファイル）

`{カレントディレクトリ}/tmp/review/review_{pr_number}_{title}.md` に以下の形式で出力する。

```markdown
# PRレビュー #{number}: {title}

**URL**: {url}
**作成日時**: {datetime}

---

## PR概要

（PR全体の概要・変更目的・全体的な所感を記載）

---

## 未解決コメントへの対応方針

（未解決コメントがない場合はこのセクションを省略する）

### {ファイルパス}:{行}

> @{author}: {元コメント内容}

- **対応要否**: 対応必要
- **対応方針**: {方針の説明}

**返信内容:**

{提案（対応方針）を先に書き、次に理由・背景を続ける。例：「〜という対応をしようと思いますが、いかがでしょうか？ 〜という背景があるためです。」}

---

### {ファイルパス}:{行}

> @{author}: {元コメント内容}

- **対応要否**: 対応不要
- **理由**: {理由}

**返信内容:**

{対応不要の結論を先に書き（「〜はスルーで大丈夫だと思いますが、いかがでしょうか？」）、次に理由を続ける。例：「この指摘はスルーで大丈夫だと思いますが、いかがでしょうか？ 〜のためです。」}

---

## 新規指摘事項

（指摘がない場合は「特に指摘事項はありませんでした。」と記載）
（修正を強く促すものから先に、確認・任意の指摘を後に並べる。**重要度ラベルは付けない**）

### {指摘の概要}

- **ファイル**: {ファイルパス}
- **行**: {行数}（必須。特定できない場合は `-`）

**指摘内容:**

{コメント内容（言い回しでトーンを伝える。ラベルは書かない）}

（コードスニペットがある場合は ```suggestion ... ``` ブロックで提案を囲む。**変更行のみ**を含め、前後の未変更行は入れない）

---

### {指摘の概要}

- **ファイル**: {ファイルパス}
- **行**: {行数}（必須。特定できない場合は `-`）

**指摘内容:**

{コメント内容}
```
