# フェーズ2：差分調査（サブエージェント用詳細指示）

サブエージェントはこのファイルの指示に厳密に従い、指定フォーマットの返り値だけを返す。
**入力（フェーズ1のノードID一覧・対応関係）は本プロンプトの冒頭に添付されている。それを使うこと。**

冒頭に添付されたノードID一覧をもとに、全フレームのBefore/After差分を調査して返してください。
説明・感想・補足は一切不要です。

## 手順

全フレームのペアに対して、BeforeとAfterそれぞれに get_design_context を実行し、
以下の項目を比較・記録する。

| 比較項目 | 確認ポイント |
|----------|-------------|
| テキスト内容 | 追加・削除・変更 |
| テキストカラー | カラーコードで比較（例: #888888 → #282828） |
| フォントサイズ・ウェイト | 数値で記録 |
| 余白・レイアウト | padding / margin / gap の変化 |
| 要素の表示条件 | 条件付き表示の変化 |
| data-development-annotations | デザイナーの実装向け注記。必ず確認する |

## 厳守ルール
- ツール呼び出しをスキップしてはいけない
- 1画面でも省略した場合、差分の洗い出しは完了とみなさない
- 変更なしの画面も「変更なし」として表に明記すること

## 返り値フォーマット（このフォーマット以外を返してはいけない）

【差分一覧】
| 画面名 | 変更箇所 | Before | After | デザイナー注記 |
|--------|---------|--------|-------|--------------|

【処理済みノードID】
| 画面名 | Before ノードID | After ノードID |
|--------|---------------|--------------|
