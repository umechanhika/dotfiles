### 使い方

```
/figma-diff-to-imple {Figma URL}
```

### 特徴

- Figma MCPのツール(`get_metadata` / `get_screenshot` / `get_design_context`)を利用して、デザイン変更点を網羅的に洗い出します。
- 各作業はサブエージェントで実行するため、メインのコンテキストを汚染しません。
- リポジトリから技術スタックを特定して、実装計画を作成します。

### Figma側の推奨対応

これらを実施していると、AIエージェントがMCP経由で効率的に差分を読み込むことができます。

- `Before/After × 画面フロー`でセクションを分ける
- 各デザインノード（画面デザイン）は`画面名 / 状態`などそのデザインノード固有の状態を表すように命名する
- Dev ModeのAnnotation機能で変更内容を明記する
