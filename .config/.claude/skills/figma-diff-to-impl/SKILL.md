# Figma差分 → 実装計画スキル

FigmaのBefore/Afterデザインを比較し、リポジトリの技術スタックに合わせた実装計画を作成する。

## アーキテクチャ方針

**全フェーズをサブエージェントに委譲し、メインセッションには圧縮された成果物だけを返す**（メインのコンテキスト消費を最小化）。設計意図とフェーズ→成果物の対応図は `references/architecture.md` 参照。

### サブエージェントの起動方法

各フェーズは Claude Code の **`Task` ツール**で起動する。専用の定義ファイルは不要。
**各フェーズの詳細な指示（作業手順＋返り値フォーマット）は `references/phaseN.md` に置いてあり、サブエージェント自身に Read させる**。これによりメインセッションのコンテキストに詳細プロンプトを載せずに済む。Figma MCP を含む利用可能なツールはサブエージェントに自動的に引き継がれる。

メインは次の手順で各フェーズを起動する:

1. スキルの絶対パスを解決する（`echo ${CLAUDE_SKILL_DIR}` を一度実行）。以後 `<SKILL_DIR>` と表記。
2. 次の形のプロンプトで `Task` を起動する:

```
<SKILL_DIR>/references/phaseN.md を Read し、その指示に厳密に従って作業し、指定された返り値フォーマットだけを返してください。

# 入力（前フェーズの返り値）
（フェーズ1以降のみ：前フェーズの返り値をそのまま貼る。フェーズ1には対象 Figma URL も添える）
```

フェーズは必ず**順次実行**すること（前フェーズの返り値を次フェーズに渡すため、並列実行は不可）。

---

## メインセッションの進行手順

以下の順で `Task` ツールを使いサブエージェントを順次起動する。各フェーズの詳細指示は対応する `references/phaseN.md` をサブエージェントに Read させる（上記「サブエージェントの起動方法」のプロンプト形を使う）。前フェーズの返り値は次フェーズの Task プロンプト冒頭に貼って引き継ぐこと。メインは各フェーズの返り値（指定フォーマット）だけを受け取り、内容を読み込まずに次フェーズへ渡す。

---

### フェーズ 0：技術スタック判定エージェント

- **前提**: なし（最初のフェーズ）。
- **詳細指示**: `references/phase0.md`（作業手順＋返り値フォーマット）をサブエージェントに Read させる。
- **返り値**: 技術スタック判定結果（数行）。

---

### フェーズ 1：構造把握エージェント

- **前提**: フェーズ0の返り値と対象 Figma URL を Task プロンプト冒頭に添えて渡す。
- **詳細指示**: `references/phase1.md` をサブエージェントに Read させる。
- **返り値**: ノードID一覧・Before/After 対応関係の表。

---

### フェーズ 2：差分調査エージェント

- **前提**: フェーズ1の返り値（ノードID一覧・対応関係）を Task プロンプト冒頭に添えて渡す。
- **詳細指示**: `references/phase2.md` をサブエージェントに Read させる。
- **返り値**: 差分一覧・処理済みノードID の表。

---

### フェーズ 3：実装計画エージェント

- **前提**: フェーズ0（技術スタック判定）とフェーズ2（差分一覧）の返り値を Task プロンプト冒頭に添えて渡す。
- **詳細指示**: `references/phase3.md` をサブエージェントに Read させる。
- **返り値**: 変更対象ファイル一覧・変更内容詳細・再利用ユーティリティ・検証方法。

---

## メインセッションの最終出力

フェーズ3の返り値をそのままユーザーに提示する。
必要に応じてフェーズ2の差分一覧表も添える。

---

## モジュールマップ

各フェーズの詳細指示はサブエージェントが Read する（メインは読まない）。

- 設計意図・フェーズ構成図 → `references/architecture.md`
- フェーズ0（技術スタック判定）の詳細指示 → `references/phase0.md`
- フェーズ1（構造把握）の詳細指示 → `references/phase1.md`
- フェーズ2（差分調査）の詳細指示 → `references/phase2.md`
- フェーズ3（実装計画）の詳細指示 → `references/phase3.md`
