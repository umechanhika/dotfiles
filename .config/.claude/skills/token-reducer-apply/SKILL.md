# Token Reducer Apply

`/token-reducer` の段階2で提示された実行計画を、実際のファイル編集として適用する。

## 前提チェック（フォールバック禁止）

直前の `/token-reducer` 段階2 で提示された実行計画が会話 context に存在することを前提とする。
存在しない場合は**推測で進めず**、以下を報告して停止する:

> 適用対象の実行計画が会話に見当たりません。先に `/token-reducer` を実行してください。

## 適用手順

1. **reference.md を読む**：`${CLAUDE_SKILL_DIR}/../token-reducer/reference.md`（apply_log 書式・保存先・効果検証手順）。

2. **実行計画を確認**：会話 context の実行計画から、適用順序・トークンインパクト順・破壊的変更の有無を把握する。

3. **適用（1ファイルずつ）**：以下のメカニズムを対象とする:
   - hook（`settings.json` の hooks セクション）
   - settings.json（MCP/プラグイン無効化・既定固定）
   - skill 本体（SKILL.md）・参照ファイル・スクリプトの新規作成や分割（観点13・14）
   - サブエージェント定義、CLAUDE.md（スリム化・移譲）、LSP プラグイン設定
   - **観点14のモジュール分割**は、分割とモジュールマップ（責務→ファイル）を常にセットで適用する（マップなし分割は次回全モジュール読みを招き削減が消える）。

4. **破壊的変更は個別確認**：「ファイルを上書き/削除」「hook 追加による既存挙動変更」等は適用前にユーザーへ確認する。

5. **apply_log 追記**：適用後、変更内容と本日の日付を apply_log に追記する（書式・保存先は reference.md）。

6. **効果検証の案内**：「効果は新セッションが溜まってから `--split-date <適用日>` で縦断比較できます。手順は reference.md を参照してください。」と案内する。

## 禁止事項

- 実行計画に無い変更を推測で行わない（フォールバック禁止）。
- **履歴は削除しない**（横断監査の自滅＝次回データ枯渇・縦断検証不能・`--resume` 不可・監査証跡の不可逆喪失を招く）。
