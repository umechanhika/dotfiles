# Token Reduction Review

現在のセッションのトークン消費を振り返り、**次回以降の消費を減らす改善を最大1件**提案する。
**提案を出力するだけ**で、変更の適用はしない（適用はユーザーが適切なリポジトリで自分で行う）。

## 手順

1. **ダイジェスト取得**（モデルを使わない決定論的解析）:
   ```bash
   python3 ~/.claude/skills/token-reduction-review/token-digest.py
   ```
   現在のセッションの transcript を自動特定し、digest(JSON) を出力する。主な項目:
   - `peak_context_tokens` 到達した最大コンテキスト / `tool_output_bytes` ツール出力累積 /
     `top_tool_output`・`top_file_reads` 消費の大きい順 / `mcp_tools` MCP 使用状況 /
     `tools` 各ツールの呼び出し回数・出力量。
2. digest から **毎セッション構造的に繰り返す** 消費の大きい箇所を特定する。
3. 候補を **品質ゲート**（下記）で自己審査する。digest で足りない時だけ transcript を
   **部分的に** Read する（全読みしない＝自分の消費を増やさない）。
4. 通過した候補のうち「削減量 × 実行可能性」が最大の **1件だけ** を出力する。
   一つも通過しなければ「特になし」と述べて終える（それが正しい挙動）。

## 品質ゲート（全て満たさなければ提案しない）

- **再現性が最重要**: そのセッション固有の偶発ではなく、毎セッション構造的に繰り返す無駄であること。
  満たさなければ削減量が大きくても見送る。
- **継続的な削減**: 毎ターン再課金される変動コンテキストや、恒久的に効く削減であること。単発で数百トークンは対象外。
- **具体的に実行可能**: 「コンテキストを減らせ」ではなく、対象ファイル/行/フック/コマンドまで落ちていること。
  適用できるコード片・設定スニペットのドラフトを含める。

## 手段カタログ（機械処理できるものは LLM 判断より決定論的スクリプト/CLI を優先）

- 前処理スクリプト/CLI 化: 繰り返し読む大出力（ログ/テスト/JSON/ビルド）を grep/jq/awk や
  専用 CLI で抽出・要約してから渡す（PreToolUse hook の出力フィルタを含む）。
- 使い捨て抽出の常設化: その場限りの処理を、再利用可能なスクリプトへ昇格。
- MCP→CLI 置換: 重い MCP サーバーを、より軽い公式 CLI（gh/aws/gcloud 等）で代替できないか。
- コンテキスト構成の是正: CLAUDE.md のスリム化・skill 移譲、不要 MCP/skill の無効化。
- 隔離: 冗長操作のサブエージェント委譲。
- 出力削減: effort レベル・思考予算（MAX_THINKING_TOKENS）の調整。
- ワークフロー: 曖昧プロンプトの具体化、plan mode の活用。

## スコープと適用先（出力に必ず明記）

- **リポジトリ固有**（このプロジェクトの CLAUDE.md / `.claude/settings.json` / scripts / 特定 MCP 等）
  → このリポジトリで適用すべき提案。
- **グローバル**（全プロジェクト共通の癖、ユーザーレベルの設定/運用）
  → ユーザー設定（`~/.claude`、実体は dotfiles）で適用すべき提案。
- 提案には「**どこで適用すべきか**（このリポジトリ / dotfiles 等）」を必ず書く。**この場では適用しない**。

## 出力フォーマット（提案が1件ある場合）

> ## 何が無駄か（診断）
> digest の数値を引用し、毎セッション繰り返す構造的消費を1点だけ説明する。
>
> ## 改善案（そのまま適用できる形）
> 適用するファイル/設定と、コード片・設定スニペットのドラフト。
>
> ## 適用先 / スコープ
> repo（このリポジトリ）or global（dotfiles）。どこで実行すべきか。
>
> ## 削減の検証方法
> 適用後にどう測れば効果を確認できるか（/context, /usage 等）。

## 厳守

- **提案を出力するだけ**。ファイル変更・コマンド実行による適用は一切しない。
- **他リポジトリを見に行ったり書き換えたりしない**（cwd 外への操作は承認コストと文脈混乱を生むため）。
- 提案は最大1件。価値ある提案が無ければ「特になし」と述べる。
- 最後に一言添える: 「適用は適切なリポジトリで行ってください。セッションを終えるなら `/exit` か `Ctrl-D`」。
